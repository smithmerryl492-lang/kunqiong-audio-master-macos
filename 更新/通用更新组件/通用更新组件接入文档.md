# 通用软件更新组件 (Universal Updater) 接入指南

本组件是一个独立、通用的 Windows 软件自动更新工具 (`updater.exe`)。它设计用于解决**任何**语言（Python, C#, C++, Node.js/Electron 等）开发的 Windows 桌面应用在自我更新时面临的文件占用、进程冲突和 UI 交互问题。

## 核心特性

*   **独立进程**：完全独立于主程序运行，确保主程序文件可被覆盖。
*   **零依赖接入**：通过标准命令行参数调用，无需链接任何库。
*   **全流程接管**：支持 **下载 -> 校验 -> 等待主进程退出 -> 解压覆盖 -> 重启主程序** 的完整流程。
*   **美观 UI**：内置基于 PySide6 的现代化进度面板，支持高分屏。
*   **进程守护**：智能等待并强制结束主程序进程，防止更新失败。

## 快速开始

### 1. 部署
将 `updater.exe` 放置在您的软件安装目录中，通常与主程序（例如 `MyApp.exe`）同级。

```text
InstallationDir/
├── MyApp.exe       (您的主程序)
├── updater.exe     (本通用更新组件)
└── ...
```

### 2. 调用协议
当您的主程序检测到需要更新时，只需构造一条命令启动 `updater.exe`，然后**立即退出**即可。

**最简调用示例（推荐）：**
让 Updater 负责下载，用户体验最好。
```bash
updater.exe --url "https://example.com/update.zip" --hash "sha256_value" --dir "C:\AppPath" --exe "MyApp.exe" --pid 1234
```

## 命令行参数详解

| 参数 | 必填 | 说明 |
| :--- | :--- | :--- |
| `--dir` | **是** | 软件的安装根目录（更新包将解压到此位置）。通常传入当前运行目录。 |
| `--exe` | **是** | 更新完成后需要重启的主程序文件名（如 `MyApp.exe`）。 |
| `--pid` | 否 | 主程序的进程 ID。传入后，Updater 会等待该进程结束，超时则强制结束它。**强烈推荐传入**。 |
| `--url` | 否 | 更新包（ZIP）的下载链接。若传入，Updater 会显示下载进度。 |
| `--hash` | 否 | 更新包的 SHA256 哈希值。若传入，Updater 下载后会自动校验，不一致则报错。 |
| `--zip` | 否 | 本地更新包的绝对路径。如果您选择在主程序中下载好更新包，则使用此参数替代 `--url`。 |

**注意**：`--url` 和 `--zip` 必须二选一。

## 多语言接入示例

### 1. Python (PyQt / PySide / Tkinter)

```python
import sys
import os
import subprocess

def start_update(download_url, file_hash):
    updater_path = os.path.join(os.getcwd(), "updater.exe")
    main_exe = "MyApp.exe"
    pid = os.getpid()
    
    # 构建命令
    cmd = [
        updater_path,
        "--url", download_url,
        "--hash", file_hash,
        "--dir", os.getcwd(),
        "--exe", main_exe,
        "--pid", str(pid)
    ]
    
    # 关键：使用 Popen 启动独立进程
    # creationflags=0x00000008 (DETACHED_PROCESS) 确保主程序退出后 updater 不会被连带关闭
    subprocess.Popen(cmd, creationflags=subprocess.CREATE_NO_WINDOW | subprocess.DETACHED_PROCESS)
    
    # 立即退出主程序
    sys.exit(0)
```

### 2. C# (.NET Core / Framework / WPF / WinForms)

```csharp
using System.Diagnostics;
using System.IO;

public void StartUpdate(string url, string hash)
{
    string updaterPath = Path.Combine(AppDomain.CurrentDomain.BaseDirectory, "updater.exe");
    string appDir = AppDomain.CurrentDomain.BaseDirectory;
    string exeName = "MyApp.exe";
    int pid = Process.GetCurrentProcess().Id;

    ProcessStartInfo startInfo = new ProcessStartInfo(updaterPath);
    startInfo.Arguments = $"--url \"{url}\" --hash \"{hash}\" --dir \"{appDir}\" --exe \"{exeName}\" --pid {pid}";
    startInfo.UseShellExecute = true; // 必须为 true 以独立启动
    startInfo.CreateNoWindow = true;

    try 
    {
        Process.Start(startInfo);
        Environment.Exit(0); // 立即退出
    }
    catch (Exception ex)
    {
        // 处理启动失败
    }
}
```

### 3. Node.js (Electron)

```javascript
const { spawn } = require('child_process');
const path = require('path');
const { app } = require('electron');

function startUpdate(url, hash) {
    const updaterPath = path.join(process.resourcesPath, 'updater.exe'); // 根据实际打包路径调整
    const appDir = path.dirname(process.execPath);
    const exeName = 'MyApp.exe';
    
    const args = [
        '--url', url,
        '--hash', hash,
        '--dir', appDir,
        '--exe', exeName,
        '--pid', process.pid
    ];

    const subprocess = spawn(updaterPath, args, {
        detached: true, // 关键：分离进程
        stdio: 'ignore'
    });

    subprocess.unref(); // 允许父进程退出而不等待子进程
    app.quit();
}
```

### 4. C++ (Qt Framework)

```cpp
#include <QProcess>
#include <QCoreApplication>
#include <QDir>

void startUpdate(const QString &url, const QString &hash) {
    QString updater = QCoreApplication::applicationDirPath() + "/updater.exe";
    QStringList args;
    args << "--url" << url
         << "--hash" << hash
         << "--dir" << QCoreApplication::applicationDirPath()
         << "--exe" << "MyApp.exe"
         << "--pid" << QString::number(QCoreApplication::applicationPid());

    // startDetached 是关键，确保 updater 独立运行
    if (QProcess::startDetached(updater, args)) {
        QCoreApplication::quit();
    }
}
```

## 最佳实践与注意事项

1.  **文件结构要求**：更新包（ZIP）内部应当直接包含文件，**不要**包含顶层文件夹。
    *   ✅ 正确：ZIP -> `MyApp.exe`, `lib.dll`
    *   ❌ 错误：ZIP -> `MyAppFolder/` -> `MyApp.exe`
2.  **Updater 自身的更新**：
    *   通常 `updater.exe` 逻辑稳定，不需要频繁更新。
    *   如果必须更新 `updater.exe`，建议在 ZIP 包中将其重命名（如 `updater_new.exe`），并在主程序下次启动时替换。
3.  **权限**：
    *   如果软件安装在 `C:\Program Files` 等受保护目录，`updater.exe` 启动时可能需要申请管理员权限（UAC）。本组件已包含 `requireAdministrator` 清单，通常会自动请求提升权限。
