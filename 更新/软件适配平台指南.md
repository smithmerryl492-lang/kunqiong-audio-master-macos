## 一、平台概述

### 1.1 平台功能

本平台是一个软件版本管理与自动更新系统，为软件提供：

| 功能 | 说明 |
|------|------|
| **安装包分发** | 新用户首次下载安装 |
| **自动更新** | 老用户自动检测并下载更新 |
| **版本管理** | 多版本并存，支持回滚 |
| **强制更新** | 可设置关键版本强制用户升级 |
| **更新统计** | 查看用户的更新行为数据 |

### 1.2 更新模式

本平台采用**全量更新**模式：

```
┌─────────────────────────────────────────────────────────────┐
│                     全量更新流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  客户端(1.0.0)          平台服务器           更新包内容      │
│       │                      │                      │       │
│       │  1. 检查更新         │                      │       │
│       ├──────────────────────>                      │       │
│       │                      │                      │       │
│       │  2. 返回更新信息     │                      │       │
│       │  (版本2.0.0)         │                      │       │
│       <──────────────────────┤                      │       │
│       │                      │                      │       │
│       │  3. 下载完整包       │                      │       │
│       ├─────────────────────────────────────────────>       │
│       │                      │                      │       │
│       │  4. 备份当前版本     │                      │       │
│       │  (1.0.0 → backup)   │                      │       │
│       │                      │                      │       │
│       │  5. 解压覆盖文件     │                      │       │
│       │  (update.zip → app) │                      │       │
│       │                      │                      │       │
│       │  6. 更新完成         │                      │       │
│       │  (当前版本 2.0.0)   │                      │       │
│       │                      │                      │       │
└─────────────────────────────────────────────────────────────┘
```

## 二、更新机制说明

### 2.1 安装包 vs 更新包

| 类型 | 用途 | 目标用户 | 文件格式 |
|------|------|----------|----------|
| **安装包** | 首次安装软件 | 新用户 | `.exe` `.msi` `.dmg` `.apk` 等 |

### 2.3 关键特性

1. **自动回滚**：更新失败时自动恢复到上一版本
2. **文件校验**：SHA256 哈希值验证文件完整性
3. **断点续传**：支持下载中断后继续下载
4. **强制更新**：关键版本可强制用户升级

## 四、更新包制作规范

### 4.1 更新包结构

更新包是一个 **ZIP 压缩文件**，其内部结构必须与你的程序目录结构一致：

```
示例：假设你的程序安装在 C:\Program Files\MyApp\

程序目录结构:
C:\Program Files\MyApp\
├── MyApp.exe              <- 主程序
├── config.json            <- 配置文件
├── resources\
│   ├── logo.png
│   └── styles.css
└── plugins\
    └── plugin1.dll

对应的更新包结构 (update_v2.0.0.zip):
update_v2.0.0.zip
├── MyApp.exe              <- 新版本主程序
├── config.json            <- 新配置文件
├── resources\
│   ├── logo.png           <- 更新后的资源
│   └── styles.css
└── plugins\
    └── plugin1.dll        <- 更新的插件

⚠️ 重要:
- 更新包解压后直接覆盖到程序目录
- 不要包含外层文件夹（不要 MyApp/MyApp.exe）
- 保持与安装目录一致的文件结构
```

### 4.2 文件组织规则

```
┌─────────────────────────────────────────────────────────────────────┐
│                     更新包应该包含的文件                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ✅ 应该包含：                                                       │
│  ├─ 所有修改过的可执行文件（.exe .dll .so等）                        │
│  ├─ 更新的资源文件（图片、样式、脚本等）                             │
│  ├─ 新增的功能模块                                                   │
│  └─ 默认配置文件（仅默认值，不包含用户自定义配置）                    │
│                                                                     │
│  ❌ 不应该包含：                                                     │
│  ├─ 用户数据文件（用户文档、设置、数据库等）                         │
│  ├─ 日志文件                                                         │
│  ├─ 临时文件                                                         │
│  ├─ 缓存文件                                                         │
│  └─ 其他运行期生成的文件                                             │
│                                                                     │
│  ⚠️ 特别注意：                                                       │
│  ├─ 如果配置文件有格式变更，提供配置迁移逻辑                         │
│  ├─ 不要删除用户数据目录                                             │
│  └─ 考虑向后兼容性                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.3 更新包命名规范

```
推荐命名格式: {软件标识}_v{版本号}_update.zip

示例:
┌──────────────────────────────────────┬─────────────────────────────┐
│ 文件名                               │ 说明                        │
├──────────────────────────────────────┼─────────────────────────────┤
│ myapp-windows_v2.1.0_update.zip      │ MyApp Windows版 2.1.0 更新  │
│ myapp-macos_v2.1.0_update.zip        │ MyApp Mac版 2.1.0 更新      │
│ toolbox_v1.5.0-beta_update.zip       │ 工具箱 1.5.0 测试版更新     │
└──────────────────────────────────────┴─────────────────────────────┘
```

### 4.4 创建更新包的步骤

**方法一：手动打包**

```powershell
# Windows PowerShell 示例
# 假设新版本文件都在 D:\Build\MyApp_v2.0.0\ 目录下

# 1. 进入构建目录
cd D:\Build\MyApp_v2.0.0

# 2. 创建ZIP包（注意：直接在目录内创建，不包含外层文件夹）
Compress-Archive -Path * -DestinationPath D:\Release\myapp_v2.0.0_update.zip

# 3. 验证包内容
Expand-Archive -Path D:\Release\myapp_v2.0.0_update.zip -DestinationPath D:\Test\temp
# 检查 D:\Test\temp 内的文件结构是否正确
```

**方法二：使用构建脚本**

```python
#!/usr/bin/env python
# create_update_package.py
import zipfile
import os
import sys

def create_update_package(source_dir, version, output_dir="."):
    """
    创建更新包
    
    Args:
        source_dir: 新版本文件所在目录
        version: 版本号（如 2.0.0）
        output_dir: 输出目录
    """
    if not os.path.exists(source_dir):
        print(f"错误: 源目录不存在 {source_dir}")
        return False
    
    # 生成文件名
    package_name = f"myapp_v{version}_update.zip"
    output_path = os.path.join(output_dir, package_name)
    
    # 确保输出目录存在
    os.makedirs(output_dir, exist_ok=True)
    
    # 创建ZIP包
    with zipfile.ZipFile(output_path, 'w', zipfile.ZIP_DEFLATED) as zf:
        for root, dirs, files in os.walk(source_dir):
            for file in files:
                file_path = os.path.join(root, file)
                # 计算相对路径（不包含 source_dir 本身）
                arcname = os.path.relpath(file_path, source_dir)
                zf.write(file_path, arcname)
                print(f"  添加: {arcname}")
    
    # 计算文件大小
    file_size = os.path.getsize(output_path)
    print(f"\n✅ 更新包创建成功!")
    print(f"   文件: {output_path}")
    print(f"   大小: {file_size / 1024 / 1024:.2f} MB")
    
    return output_path

# 使用示例
if __name__ == "__main__":
    # 从命令行参数获取
    if len(sys.argv) >= 3:
        source = sys.argv[1]
        version = sys.argv[2]
    else:
        # 默认配置
        source = input("新版本文件目录: ").strip().strip('"')
        version = input("版本号 (如 2.0.0): ").strip()
    
    create_update_package(source, version, "./release")
```

### 4.5 安装包制作（新用户使用）

安装包是给新用户首次安装时使用的：

| 平台 | 安装包格式 | 制作工具 |
|------|-----------|----------|
| Windows | `.exe` `.msi` | Inno Setup, NSIS, WiX |
| macOS | `.dmg` `.pkg` | Packages, dmgcanvas |
| Linux | `.deb` `.rpm` `.AppImage` | 各发行版工具 |
| Android | `.apk` `.aab` | Android Studio |
| iOS | 通过 App Store | Xcode |

**安装包 vs 更新包的区别：**

```
安装包 (Setup.exe):
├── 安装向导界面
├── 选择安装目录
├── 创建快捷方式
├── 写入注册表/系统配置
├── 安装运行环境（如 .NET Runtime）
└── 首次启动配置

更新包 (update.zip):
├── 仅包含程序文件
├── 直接覆盖安装目录
├── 保留用户数据和配置
└── 不包含安装逻辑
```

### 5.2 AutoUpdater 初始化参数详解

在使用 `AutoUpdater` 之前，你需要准备以下四个参数：

#### 参数一：server_url（服务器地址）

**作用**：指定更新平台的服务器地址

**格式**：
```python
# 开发环境（本地测试）
server_url = 'http://127.0.0.1:8000'

# 局域网环境
server_url = 'http://192.168.1.100:8000'

# 生产环境（域名+HTTPS）
server_url = 'https://update.yourcompany.com'
```

**获取方式**：
- 开发测试时使用 `http://127.0.0.1:8000`
- 生产环境询问平台管理员获取正式域名

---

#### 参数二：software_id（软件标识符）

**作用**：唯一标识你的软件，平台通过此ID区分不同软件

**命名规则**：
- 只能包含小写字母、数字、下划线、连字符
- 建议格式：`软件名-平台`

**示例**：
```python
# 同一软件的不同平台版本使用不同ID
software_id = 'myapp-windows'      # Windows版
software_id = 'myapp-macos'        # macOS版  
software_id = 'myapp-linux'        # Linux版
software_id = 'myapp-android'      # Android版
```

**获取方式**：
- 联系平台管理员注册软件后获取
- 可在平台管理后台查看

**动态获取示例**（自动识别平台）：
```python
import platform

def get_software_id():
    system = platform.system().lower()
    if system == 'windows':
        return 'myapp-windows'
    elif system == 'darwin':
        return 'myapp-macos'
    elif system == 'linux':
        return 'myapp-linux'
    return 'myapp-windows'  # 默认

software_id = get_software_id()
```

---

#### 参数三：current_version（当前版本号）

**作用**：当前安装的软件版本，用于与平台最新版本比较

**格式**：遵循语义化版本规范 `MAJOR.MINOR.PATCH`
```python
current_version = '1.0.0'    # 初始版本
current_version = '2.1.3'    # 有功能更新和bug修复
current_version = '2.1.0-beta'  # 测试版
```

**获取方式**：

**方式一：从文件读取（推荐）**
```python
def get_current_version():
    """从 version.txt 读取版本号"""
    version_file = os.path.join(app_dir, 'version.txt')
    if os.path.exists(version_file):
        with open(version_file, 'r', encoding='utf-8') as f:
            return f.read().strip()
    return '1.0.0'  # 默认版本

current_version = get_current_version()
```

**方式二：从配置文件读取**
```python
import json

def get_current_version():
    """从 config.json 读取版本号"""
    config_file = os.path.join(app_dir, 'config.json')
    if os.path.exists(config_file):
        with open(config_file, 'r', encoding='utf-8') as f:
            config = json.load(f)

    return '1.0.0'
```

#### 参数四：app_dir（应用程序目录）

**作用**：指定软件的安装目录，更新包会解压到此目录进行文件覆盖

**重要说明**：
- 此目录必须可写（有写入权限）
- 会在同级目录自动创建 `backup/`（备份）和 `temp_update/`（临时文件）子目录
- 更新包内的文件会解压并覆盖到此目录下的对应位置

**示例**：

```python
class UpdateInfo:

class AutoUpdater:
    """
    自动更新器 - 集成到你的应用中
    
    使用示例:
        updater = AutoUpdater(

    ├─────────────────┼──────────┼─────────────────────────────────────────────────────────┤
    │ server_url      │ str      │ 平台服务器地址                                         │
    │                 │          │ 示例: 'http://192.168.1.100:8000' 或                    │

    │                 │          │                                                         │
    │ software_id     │ str      │ 软件唯一标识符                                         │
    │                 │          │ 由平台管理员分配，区分大小写                            │
    │                 │          │ 示例: 'myapp-desktop', 'myapp-win', 'myapp-macos'       │
    │                 │          │                                                         │
    │ current_version │ str      │ 当前安装的软件版本号                                   │
    │                 │          │ 格式: x.y.z 如 '1.0.0', '2.1.3'                         │
    │                 │          │ 用于与平台最新版本比较                                 │
    │                 │          │                                                         │
    │ app_dir         │ str      │ 应用程序的根目录路径                                   │
    │                 │          │ 更新包会解压到此目录进行文件覆盖                       │
    │                 │          │ 会在同级目录创建 backup/ 和 temp_update/ 子目录        │
    │                 │          │ 示例: 'C:\\Program Files\\MyApp'                        │
    └─────────────────┴──────────┴─────────────────────────────────────────────────────────┘
    """
    
    def __init__(self, server_url: str, software_id: str, current_version: str, app_dir: str):
        self.server_url = server_url.rstrip('/')
        self.software_id = software_id
        self.current_version = current_version
        self.app_dir = Path(app_dir)
        self.backup_dir = self.app_dir.parent / 'backup'
        self.temp_dir = self.app_dir.parent / 'temp_update'
        self.update_package_path: Optional[Path] = None
        
        self.backup_dir.mkdir(exist_ok=True)
        self.temp_dir.mkdir(exist_ok=True)
    
    def check_update(self) -> UpdateInfo:
        """检查是否有新版本"""
        try:
            url = f"{self.server_url}/api/v1/updates/check/"
            params = {
                'software': self.software_id,
                'version': self.current_version
            }
            
            logger.info(f"正在检查更新...")
            response = requests.get(url, params=params, timeout=10)
            response.raise_for_status()
            
            data = response.json()
            
            if data.get('has_update'):
                logger.info(f"发现新版本: {data.get('version')}")
                return UpdateInfo(
                    has_update=True,
                    version=data.get('version'),
                    update_log=data.get('update_log'),
                    download_url=data.get('download_url'),
                    package_size=data.get('package_size'),
                    package_hash=data.get('package_hash'),
                    is_mandatory=data.get('is_mandatory', False),
                    release_date=data.get('release_date')
                )
            else:
                logger.info("当前已是最新版本")
                return UpdateInfo(has_update=False)
                
        except Exception as e:
            logger.error(f"检查更新失败: {e}")
            return UpdateInfo(has_update=False)
    
    def download_update(self, update_info: UpdateInfo, progress_callback=None) -> bool:
        """下载更新包"""
        if not update_info.download_url:
            logger.error("下载地址为空")
            return False
        
        try:
            download_url = update_info.download_url
            if not download_url.startswith('http'):
                download_url = f"{self.server_url}{download_url}"
            
            filename = f"{self.software_id}_v{update_info.version}.zip"
            self.update_package_path = self.temp_dir / filename
            
            logger.info(f"开始下载: {filename}")
            
            # 断点续传
            headers = {}
            if self.update_package_path.exists():
                downloaded_size = self.update_package_path.stat().st_size
                if downloaded_size < (update_info.package_size or 0):
                    headers['Range'] = f'bytes={downloaded_size}-'
            
            response = requests.get(download_url, headers=headers, stream=True, timeout=30)
            response.raise_for_status()
            
            mode = 'ab' if 'Range' in headers else 'wb'
            downloaded = self.update_package_path.stat().st_size if self.update_package_path.exists() else 0
            
            with open(self.update_package_path, mode) as f:
                for chunk in response.iter_content(chunk_size=8192):
                    if chunk:
                        f.write(chunk)
                        downloaded += len(chunk)
                        if progress_callback and update_info.package_size:
                            progress_callback(downloaded, update_info.package_size)
            
            # 校验文件
            if update_info.package_hash:
                if not self._verify_hash(self.update_package_path, update_info.package_hash):
                    logger.error("文件校验失败")
                    self.update_package_path.unlink(missing_ok=True)
                    return False
            
            logger.info("下载完成")
            return True
            
        except Exception as e:
            logger.error(f"下载失败: {e}")
            return False
    
    def install_update(self) -> bool:
        """安装更新（带自动回滚）"""
        if not self.update_package_path or not self.update_package_path.exists():
            logger.error("更新包不存在")
            return False
        
        try:
            logger.info("开始安装更新...")
            
            # 1. 备份
            logger.info("步骤1: 备份当前版本...")
            backup_path = self._create_backup()
            if not backup_path:
                logger.error("备份失败")
                return False
            
            # 2. 解压
            logger.info("步骤2: 解压更新包...")
            extract_dir = self.temp_dir / 'extracted'
            if extract_dir.exists():
                shutil.rmtree(extract_dir)
            extract_dir.mkdir(exist_ok=True)
            
            with zipfile.ZipFile(self.update_package_path, 'r') as zip_ref:
                zip_ref.extractall(extract_dir)
            
            # 3. 应用更新
            logger.info("步骤3: 应用更新...")
            self._apply_update(extract_dir)
            
            # 4. 验证
            logger.info("步骤4: 验证更新...")
            if self._verify_update():
                logger.info("更新成功！")
                self._cleanup(extract_dir)
                return True
            else:
                raise Exception("更新验证失败")
                
        except Exception as e:
            logger.error(f"更新失败: {e}")
            logger.info("正在回滚...")
            if self._rollback():
                logger.info("回滚成功")
            else:
                logger.error("回滚失败")
            return False
    
    def _create_backup(self) -> Optional[Path]:
        """创建备份"""
        try:
            import time
            timestamp = hashlib.md5(str(time.time()).encode()).hexdigest()[:8]
            backup_name = f"{self.software_id}_v{self.current_version}_{timestamp}"
            backup_path = self.backup_dir / backup_name
            
            shutil.copytree(self.app_dir, backup_path, ignore=shutil.ignore_patterns(
                'temp_update', 'backup', '*.log', '__pycache__', '.git'
            ))
            
            return backup_path
            
        except Exception as e:
            logger.error(f"备份失败: {e}")
            return None
    
    def _apply_update(self, extract_dir: Path):
        """应用更新"""
        for src_file in extract_dir.rglob('*'):
            if src_file.is_file():
                rel_path = src_file.relative_to(extract_dir)
                dst_file = self.app_dir / rel_path
                dst_file.parent.mkdir(parents=True, exist_ok=True)
                shutil.copy2(src_file, dst_file)
    
    def _verify_update(self) -> bool:
        """验证更新"""
        # 根据你的应用添加验证逻辑
        return True
    
    def _rollback(self) -> bool:
        """回滚到上一版本"""
        try:
            backups = sorted(self.backup_dir.iterdir(), key=lambda x: x.stat().st_mtime, reverse=True)
            
            for backup in backups:
                if backup.is_dir() and backup.name.startswith(self.software_id):
                    shutil.rmtree(self.app_dir)
                    shutil.copytree(backup, self.app_dir)
                    return True
            
            return False
            
        except Exception as e:
            logger.error(f"回滚失败: {e}")
            return False
    
    def _cleanup(self, extract_dir: Path):
        """清理临时文件"""
        try:
            if extract_dir.exists():
                shutil.rmtree(extract_dir)
            if self.update_package_path and self.update_package_path.exists():
                self.update_package_path.unlink()
        except Exception as e:
            logger.warning(f"清理失败: {e}")
    
    def _verify_hash(self, file_path: Path, expected_hash: str) -> bool:
        """校验文件哈希"""
        sha256_hash = hashlib.sha256()
        with open(file_path, 'rb') as f:
            for chunk in iter(lambda: f.read(8192), b''):
                sha256_hash.update(chunk)
        return sha256_hash.hexdigest().lower() == expected_hash.lower()
```

```typescript
            response.data.pipe(writer);
            
            await new Promise((resolve, reject) => {
                writer.on('finish', resolve);
                writer.on('error', reject);
            });
            
            // 校验哈希
            if (info.package_hash) {
                const hash = await this.calculateSha256(filePath);
                if (hash !== info.package_hash.toLowerCase()) {
                    fs.unlinkSync(filePath);
                    throw new Error('文件校验失败');
                }
            }
            
            return filePath;
        } catch (error) {
            console.error('下载失败:', error);
            return null;
        }
    }
    
    installUpdate(zipPath: string): boolean {
        try {
            // 1. 备份
            const backupPath = path.join(app.getPath('userData'), 'backup', Date.now().toString());
            this.copyDirectory(this.appPath, backupPath);
            
            // 2. 解压
            const zip = new AdmZip(zipPath);
            zip.extractAllTo(this.appPath, true);
            
            // 3. 清理
            fs.unlinkSync(zipPath);
            
            return true;
        } catch (error) {
            console.error('安装失败:', error);
            return false;
        }
    }
    
    private async calculateSha256(filePath: string): Promise<string> {
        return new Promise((resolve, reject) => {
            const hash = crypto.createHash('sha256');
            const stream = fs.createReadStream(filePath);
            stream.on('data', data => hash.update(data));
            stream.on('end', () => resolve(hash.digest('hex')));
            stream.on('error', reject);
        });
    }
    
    private copyDirectory(source: string, destination: string): void {
        fs.mkdirSync(destination, { recursive: true });
        const files = fs.readdirSync(source);
        
        for (const file of files) {
            const srcPath = path.join(source, file);
            const destPath = path.join(destination, file);
            const stat = fs.statSync(srcPath);
            
            if (stat.isDirectory()) {
                this.copyDirectory(srcPath, destPath);
            } else {
                fs.copyFileSync(srcPath, destPath);
            }
        }
    }
}

// 在主进程中使用
export async function checkForUpdates(mainWindow: BrowserWindow) {
    const updater = new AutoUpdater(
        'http://your-server.com',
        'myapp-electron',
        app.getVersion()
    );
    
    const info = await updater.checkUpdate();

        if (result.response === 0) {
```

## 六、版本号管理规范

### 6.1 语义化版本号（SemVer）

```
版本号格式: MAJOR.MINOR.PATCH[-PRERELEASE]

MAJOR - 主版本号：重大更新，可能不兼容旧版本
MINOR - 次版本号：新增功能，向后兼容
PATCH - 修订号：修复bug，向后兼容
PRERELEASE - 预发布标识（可选）: alpha, beta, rc 等

示例:
┌────────────┬─────────────────────────────────────┐
│ 版本号     │ 说明                                │
├────────────┼─────────────────────────────────────┤
│ 1.0.0      │ 首个正式版                          │
│ 1.0.1      │ 修复bug                             │
│ 1.1.0      │ 新增功能                            │
│ 1.2.0      │ 更多新功能                          │
│ 2.0.0      │ 重大更新（可能破坏兼容性）          │
│ 2.1.0      │ 2.x 的新功能                        │
│ 2.1.0-beta │ 2.1.0 的测试版                      │
│ 2.1.0-rc1  │ 2.1.0 的候选发布版1                 │
└────────────┴─────────────────────────────────────┘
```

### 6.2 版本号递增规则

```
场景1: 修复bug
当前: 1.0.0
修复: 1.0.1

场景2: 新增功能（向后兼容）
当前: 1.0.1
新增: 1.1.0

场景3: 重大更新（可能破坏兼容性）
当前: 1.5.0
重大: 2.0.0

场景4: 预发布版本
当前: 2.0.0
测试: 2.1.0-alpha → 2.1.0-beta → 2.1.0-rc1 → 2.1.0
```

### 6.3 版本号存储位置

**方式一：version.txt 文件（推荐）**

```
程序目录/
├── MyApp.exe
├── version.txt          <- 版本号写入此文件
└── config.json
```

```python
# 读取版本号
def get_version():
    version_file = os.path.join(app_dir, 'version.txt')
    if os.path.exists(version_file):
        with open(version_file) as f:
            return f.read().strip()
    return '1.0.0'
```

**方式二：嵌入可执行文件资源（Windows）**

```c

PRODUCTVERSION 2,0,0,0
FILEOS VOS_NT_WINDOWS32
FILETYPE VFT_APP
BEGIN
    BLOCK "StringFileInfo"
    BEGIN
        VALUE "FileVersion", "2.0.0"
        VALUE "ProductVersion", "2.0.0"
    END
END
```

**方式三：配置文件（config.json）**

```json
{
    "version": "2.0.0",
    "app_name": "MyApp",
    "other_config": "..."
}
```

## 八、常见问题

### Q1: 更新后用户数据会丢失吗？

**不会。** 更新包只包含程序文件，不会包含用户数据目录。更新过程只是覆盖程序文件，不会删除用户数据。

**建议做法：**
```
程序目录结构:
MyApp/
├── app/                    <- 程序文件（更新包覆盖此目录）
│   ├── MyApp.exe
│   └── resources/
├── data/                   <- 用户数据（不会被覆盖）
│   ├── settings.json
│   └── user_files/
└── version.txt
```

### Q2: 如何处理配置文件格式的变更？

**方案一：配置迁移代码**

```python
def migrate_config():
    """升级配置文件格式"""
    config_path = 'data/settings.json'
    
    with open(config_path) as f:
        config = json.load(f)
    
    # 获取当前配置版本
    config_version = config.get('version', '1.0.0')
    
    # 从 1.x 升级到 2.x
    if config_version.startswith('1.'):
        # 迁移旧配置到新格式
        config['new_field'] = config.pop('old_field', default_value)
        config['version'] = '2.0.0'
    
    with open(config_path, 'w') as f:
        json.dump(config, f)
```

**方案二：使用默认配置合并**

```python
def load_config():

    default_config = {

    }
```

```python
def update_and_restart(update_info):
    # 下载更新
    updater.download_update(update_info)
    
    # 创建更新脚本
    bat_script = f'''@echo off
timeout /t 2 /nobreak
xcopy /s /y "{temp_dir}\\extracted\\*" "{app_dir}\\"
start "" "{app_dir}\\MyApp.exe"
del "%~f0"
'''
    script_path = os.path.join(temp_dir, 'update.bat')
    with open(script_path, 'w') as f:
        f.write(bat_script)
    
    # 启动脚本并退出
    subprocess.Popen([script_path], shell=True)
    sys.exit(0)
```
